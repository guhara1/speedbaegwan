# 스피드 배관공사 웹사이트

전국 배관·누수·하수구 서비스 정적 웹사이트입니다. 그대로 웹호스팅에 올리면 바로 작동합니다.

## 폴더 구조

```
speed-plumbing/
├─ index.html                       ← 메인 페이지
├─ services/                        ← 서비스 4개 카테고리
│  ├─ nusu.html   (누수탐지·누수공사)
│  ├─ makhim.html (하수구·배관 막힘)
│  ├─ gyoche.html (수전·변기·설비 교체)
│  └─ seolbi.html (배관설비·고압세척·내시경)
├─ regions/                         ← 전국 행정구역 (총 3,136페이지)
│  ├─ seoul.html … jeju.html        ← 17개 광역시·도
│  ├─ <slug>/<시·군·구>/index.html   ← 252개 시·군·구 (행정구)
│  └─ <slug>/<시·군·구>/<행정동>.html ← 2,867개 대표 행정동
├─ data/
│  └─ districts.json                ← 행정구역 데이터(17시도·252시군구·3556행정동)
├─ assets/
│  ├─ css/style.css
│  ├─ js/main.js
│  └─ img/*.svg                     ← 시공사진 플레이스홀더 20장 + OG 이미지
├─ sitemap.xml                      ← 사이트맵 색인(sitemap index)
├─ sitemap-core.xml                 ← 메인·서비스·광역 페이지
├─ sitemap-<slug>.xml               ← 시·도별 시군구·행정동 (17개)
├─ robots.txt
└─ build.py                         ← 페이지 재생성 스크립트
```

전국 **행정구(시·군·구) 252개**와 **행정동(읍·면·동)**을 모두 페이지로 만들었습니다.
행정구역 계층은 `regions/<광역시도영문>/<시군구>/<행정동>.html` 구조이며, 한글 경로는
브라우저·검색엔진에서 자동으로 퍼센트 인코딩되어 그대로 동작합니다.

**번호로 나뉜 동은 대표 1개로 표기합니다.** 예를 들어 `창신1동·창신2동·창신3동`은
`창신동` 한 페이지로 묶여 표기·생성되며(해당 페이지 본문에 포함 동을 명시), 시·군·구
목록에도 대표 이름 하나만 노출됩니다. 반면 `종로1·2·3·4가동`처럼 가(街)형 이름은 그
자체가 하나의 행정동이므로 그대로 유지합니다. 이렇게 전체 행정동 3,556개가 **대표
2,867개** 페이지로 정리됩니다.

행정구역 명칭은 공공데이터(행정안전부 행정동 기준, 2025년 자료)에 근거한 사실 정보이며,
`data/districts.json` 하나만 교체하면 전체 페이지가 새 구역 정보로 재생성됩니다.

## ★ 배포 전 반드시 교체할 항목

`build.py` 상단 `SITE` 값을 실제 정보로 바꾼 뒤 `python3 build.py`를 실행하면
모든 페이지에 한 번에 반영됩니다. (스크립트 없이 직접 고쳐도 됩니다.)

| 항목 | 현재 placeholder |
|---|---|
| 도메인 | `https://www.speedbaegwan.co.kr` |
| 대표 전화 | `1668-0000` |
| 이메일 | `help@speedbaegwan.co.kr` |
| 대표자명 | `홍길동` |
| 사업자등록번호 | `000-00-00000` |
| 주소 | `서울특별시 ○○구 ○○로 000` |

그리고 **`assets/img/`의 SVG 20장을 실제 시공 사진(JPG/WEBP)으로 교체**하세요.
파일명을 그대로 쓰거나, 바꿨다면 HTML의 `src`도 함께 수정하면 됩니다.

### 재생성 방법

```bash
python3 build.py
```

`data/districts.json`(행정구역 데이터)과 `build.py` 상단 `SITE` 값을 바탕으로
메인·서비스·광역·시군구·행정동 전 페이지와 사이트맵을 한 번에 다시 만듭니다.
(약 3,830개 HTML, 수 초 내 완료)

## 검색 최적화(첨부 정책 반영 내역)

- **E-E-A-T**: 대표자·사업자번호·주소·연락처를 푸터와 회사소개에 명시. 실제 시공 사진과 후기로 교체하면 1차 경험 신호가 강해집니다.
- **구조화 데이터(JSON-LD)**: 전 페이지에 `Plumber`/`LocalBusiness`, 서비스 페이지에 `Service`, 지역·시군구·행정동 페이지에 해당 지역별 `LocalBusiness`, 서비스·지역 페이지에 `BreadcrumbList`, 메인에 `FAQPage` 적용.
- **내부 링크**: 광역 → 시·군·구 → 행정동 계층 링크와, 행정동 페이지의 인접 동네 링크로 크롤링 경로를 촘촘히 연결.
- **페이지 경험**: 반응형(모바일 최적화), 시맨틱 HTML, 이미지 lazy-load, 접근성(포커스·모션 감소) 반영. HTTPS는 호스팅에서 적용하세요.
- **선호 썸네일**: `og:image` 메타 지정 완료(추가로 schema.org `image`도 병행 가능).
- **사이트맵**: `sitemap.xml`(사이트맵 색인) + `sitemap-core.xml` + 시·도별 `sitemap-<slug>.xml` 17개 생성 완료 → Google Search Console에 `sitemap.xml`을 제출하세요.

## ⚠️ 대량 지역 페이지 — 반드시 읽어주세요 (도어웨이 페이지 위험)

요청에 따라 **전국 행정구·행정동 전체(약 3,800페이지)**를 생성했습니다. 각 페이지에는
지역명을 반영한 제목·본문·인접 지역 내부링크·지역별 `LocalBusiness` 구조화 데이터를 넣어
기계적 복제를 최대한 피했습니다. 다만 검색엔진 정책상 **지역명만 바꾼 유사 페이지를 대량으로
찍어내면 "도어웨이 페이지"로 간주되어 색인 제외·순위 하락 페널티**를 받을 수 있습니다.

색인 품질과 안전을 위해 아래를 권장합니다.

- **핵심 지역부터 고유 콘텐츠 채우기**: 실제 영업하는 시·군·구/동에 그 지역의 **실제
  시공사례·후기·현장 사진**을 넣으세요. 고유 콘텐츠가 있는 페이지일수록 색인·상위노출에 유리합니다.
- **범위 조절 옵션**: 전국 3,556개 동이 부담된다면 `build.py`의 `build_dongs()` 호출을 빼고
  시·군·구(행정구) 단위까지만 운영하거나, `data/districts.json`을 실제 영업 지역만 남기고
  줄여서 재생성할 수 있습니다.
- **점진적 색인**: `sitemap.xml`을 Search Console에 제출한 뒤 색인 상태를 보며
  저품질로 분류되는 페이지가 많으면 범위를 조절하세요.

## 배포 방법

정적 파일이라 아무 웹호스팅에나 업로드하면 됩니다.
- 무료: GitHub Pages, Netlify, Cloudflare Pages (폴더 통째로 드래그)
- 일반 호스팅: `index.html` 포함 전체를 `public_html/`에 FTP 업로드

배포 후 Google Search Console에 도메인 등록 → `sitemap.xml` 제출하면 색인이 시작됩니다.
